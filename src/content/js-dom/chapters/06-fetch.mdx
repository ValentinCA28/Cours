## C'est quoi fetch ?

<ic>fetch()</ic> est une fonction native du navigateur qui permet d'envoyer des **requêtes HTTP** (GET, POST, PUT, DELETE...) vers un serveur, sans recharger la page. C'est la base de toute application web moderne qui communique avec une API.

`fetch()` retourne une **Promise** — un objet qui représente une valeur qui arrivera **plus tard** (quand le serveur aura répondu). Il y a deux façons de gérer les Promises : `.then()/.catch()` et `async/await`.

## Syntaxe avec .then / .catch

```js
fetch("https://jsonplaceholder.typicode.com/users")
  .then(function (response) {
    // response.json() retourne aussi une Promise
    return response.json();
  })
  .then(function (data) {
    // data contient les données JSON du serveur
    console.log(data);
  })
  .catch(function (error) {
    // Gestion des erreurs réseau
    console.error("Erreur réseau :", error);
  });
```

Cette syntaxe fonctionne, mais elle peut devenir difficile à lire quand tu enchaînes plusieurs opérations.

## Syntaxe async / await <Badge>recommandé</Badge>

La syntaxe <ic>async/await</ic> est la manière moderne et plus lisible d'écrire du code asynchrone. Le mot-clé <ic>await</ic> **met en pause** l'exécution de la fonction jusqu'à ce que la Promise soit résolue.

```js
async function loadUsers() {
  try {
    const response = await fetch("https://jsonplaceholder.typicode.com/users");

    // Vérifier que la requête a réussi
    if (!response.ok) {
      throw new Error(`Erreur HTTP : ${response.status}`);
    }

    const users = await response.json();

    // Afficher les utilisateurs dans le DOM
    const list = document.querySelector("#user-list");
    list.innerHTML = "";

    users.forEach(function (user) {
      const li = document.createElement("li");
      li.textContent = `${user.name} — ${user.email}`;
      list.appendChild(li);
    });
  } catch (error) {
    console.error("Impossible de charger les utilisateurs :", error);
  }
}

// Appeler la fonction
loadUsers();
```

<Callout variant="info" title="Règle importante">
  Le mot-clé `await` ne peut être utilisé qu'à l'intérieur d'une fonction déclarée avec `async`. Si tu essaies de l'utiliser en dehors, tu auras une erreur de syntaxe.
</Callout>

## Envoyer des données avec POST

Pour envoyer des données à un serveur (créer un utilisateur, soumettre un formulaire...), tu utilises une requête **POST** avec des options supplémentaires :

```js
async function createPost(title, body) {
  try {
    const response = await fetch("https://jsonplaceholder.typicode.com/posts", {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        title: title,
        body: body,
        userId: 1,
      }),
    });

    if (!response.ok) {
      throw new Error(`Erreur HTTP : ${response.status}`);
    }

    const newPost = await response.json();
    console.log("Article créé :", newPost);
    return newPost;
  } catch (error) {
    console.error("Erreur lors de la création :", error);
  }
}

// Utilisation
createPost("Mon titre", "Le contenu de mon article");
```

Les points essentiels pour une requête POST :
- **`method: "POST"`** — indique le type de requête
- **`headers`** — précise qu'on envoie du JSON avec `"Content-Type": "application/json"`
- **`body`** — les données à envoyer, converties en chaîne JSON avec <ic>JSON.stringify()</ic>

## Attention : fetch ne lance pas d'erreur sur les codes HTTP

<Callout variant="warn" title="Piège classique : fetch et les erreurs HTTP">
  `fetch()` ne lance **pas** d'erreur quand le serveur répond avec un code d'erreur (404, 500, etc.). La Promise est résolue dès que le serveur **répond**, quel que soit le code HTTP. Seule une **erreur réseau** (pas de connexion, serveur injoignable) provoque un rejet de la Promise.

  C'est pour ça que tu dois **toujours** vérifier `response.ok` (qui vaut `true` pour les codes 200-299).
</Callout>

```js
async function fetchData(url) {
  try {
    const response = await fetch(url);

    // ❌ Sans cette vérification, un 404 passerait inaperçu !
    if (!response.ok) {
      throw new Error(`Le serveur a répondu ${response.status}`);
    }

    const data = await response.json();
    return data;
  } catch (error) {
    // Ici on attrape SOIT une erreur réseau, SOIT notre erreur HTTP
    console.error("Erreur :", error.message);
  }
}
```

## Les codes HTTP courants

| Code | Signification |
|---|---|
| **200** | OK — tout s'est bien passé |
| **201** | Created — ressource créée avec succès |
| **400** | Bad Request — la requête est mal formée |
| **401** | Unauthorized — authentification requise |
| **403** | Forbidden — accès refusé |
| **404** | Not Found — la ressource n'existe pas |
| **500** | Internal Server Error — erreur côté serveur |

## Exemple complet : recherche d'utilisateurs

```js
const searchInput = document.querySelector("#search");
const resultsList = document.querySelector("#results");

searchInput.addEventListener("input", async function () {
  const query = searchInput.value.trim();

  if (query.length < 2) {
    resultsList.innerHTML = "";
    return;
  }

  try {
    const response = await fetch(
      `https://jsonplaceholder.typicode.com/users`
    );

    if (!response.ok) {
      throw new Error(`Erreur : ${response.status}`);
    }

    const users = await response.json();

    // Filtrer côté client
    const filtered = users.filter(function (user) {
      return user.name.toLowerCase().includes(query.toLowerCase());
    });

    // Mettre à jour le DOM
    resultsList.innerHTML = "";
    filtered.forEach(function (user) {
      const li = document.createElement("li");
      li.textContent = `${user.name} (${user.email})`;
      resultsList.appendChild(li);
    });
  } catch (error) {
    resultsList.innerHTML = "<li>Erreur de chargement</li>";
    console.error(error);
  }
});
```
